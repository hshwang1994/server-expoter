# Cycle 2026-04-30 — HTTP 406 호환 fix + reverse regression hotfix + auth fallback

## Context

사용자 사이트 운영 중 발견된 다중 외부 호환성 사고:

1. **사이트별 Lenovo BMC가 "Redfish 미지원"으로 false-positive abort**
   - `curl -k -m 15 https://<bmc>/redfish/v1/` 정상 응답
   - 같은 Jenkins agent + 같은 Python에서 Python urllib 만 fail
2. **Dell host envelope에서 "계정 틀림" + vault fallback 안 됨**
   - vault accounts 순서 [A,B] / [B,A] 둘 다 첫 자격 매칭 host만 OK
3. **사용자 디버깅 정보 (errors[0].detail) 찾기 어려움** — envelope `diagnosis.details`에 detail 누락

## Root Cause

### Issue 1 — HTTP 406 (precheck Stage 3 false-positive 미지원)

[common/library/precheck_bundle.py:170](../../../common/library/precheck_bundle.py) `http_get` 가 `Accept` 헤더 미명시 → urllib default = Accept 헤더 부재 → 일부 BMC 펌웨어 (HPE iLO RedfishVersion 1.17.0 등)가 `406 Not Acceptable` 거부 → probe_redfish가 "미지원" 으로 분류.

본 수집 (`redfish_gather.py:91-95` `_get`) 은 처음부터 `Accept` + `OData-Version` 명시 — **drift 누적**: precheck만 헤더 빠짐. 일관성 hook 부재.

### Issue 2 — reverse regression (Lenovo XCC reject)

처음 fix 시 `Accept` + `OData-Version` + `User-Agent` 셋 다 추가 → Lenovo XCC 일부 펌웨어가 `OData-Version` 또는 `User-Agent` 거부. 사용자 검증으로 "Accept만 있으면 모든 BMC OK" 결론.

### Issue 3 — _compute_final_status partial 오판정

[redfish_gather.py:1781](../../../redfish-gather/library/redfish_gather.py) — 1개 섹션이라도 collected에 들어가면 'partial' 반환. detect_vendor가 ServiceRoot 무인증으로 vendor + system_uri 채울 수 있어 401 자격 fail이어도 일부 섹션 collected → 'partial'. try_one_account loop의 break 가드가 'partial'을 success로 판정 → 두 번째 자격 fallback 안 됨.

**3. _compute_final_status 401/403 강제 failed** (redfish_gather.py:1781):
```python
def _compute_final_status(collected, failed, errors=None):
    clean = [s for s in collected if s not in failed]
    if errors:
        for e in errors:
            detail = (e.get('detail') or '')
            if 'HTTP 401' in detail or 'HTTP 403' in detail:
                return 'failed', clean   # partial 무효화
    if not clean: return 'failed', clean
    if failed:    return 'partial', clean
    return 'success', clean
```
