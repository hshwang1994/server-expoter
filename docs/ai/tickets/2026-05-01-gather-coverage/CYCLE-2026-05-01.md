# Cycle 2026-05-01 — 404→'not_supported' 분류 + PowerSubsystem fallback + 3채널 인프라

## Context

CYCLE-2026-04-30 fix 후 사용자가 Dell host envelope에서 noise 보고:

```
errors [2]
section: power
message: power 정보 실패: HTTP 404: Not Found
detail: null

section: network_adapters
message: networkadapters 미지원 또는 실패: HTTP 404: Not Found
```

→ **HTTP 404 = "endpoint 자체 부재 = capability 미지원"** 인데 코드가 'failed' 분류 → errors[] 노이즈.

사용자 명시: "권장으로 진행 + 모든 섹션 + OS/ESXi 까지" — 3채널 전체 확장.

## Root Cause

### 1. 404 분류 결함 (모든 vendor 공통)

[redfish_gather.py:1732 _make_section_runner](../../../redfish-gather/library/redfish_gather.py) — section 함수가 errors 반환 시 무조건 `all_errors.extend(errs)` + `failed.append(section)`. **404 (endpoint 부재) vs 5xx (진짜 fail) 구분 없음**.

영향 모델:
- Dell iDRAC 8 (구 PowerEdge) — `Power` schema 미지원
- Dell PowerEdge 일부 모델 — `Chassis/{id}/NetworkAdapters` 미지원 (EthernetInterfaces만)
- Lenovo IMM2 — `Power` 미지원
- Supermicro X9 — 매우 제한
- HPE iLO 4 — `Storage` / `NetworkAdapters` 미지원

### 2. DMTF 표준 변천 (PowerSubsystem)

DMTF 2020.4 release (~Redfish 1.13)에서:
- `Power` 1.x → `PowerSubsystem` + `PowerSupply` + `PowerSupplyMetrics` (deprecated)
- `Thermal` 1.x → `ThermalSubsystem` + `Fan` + `EnvironmentMetrics` (deprecated)

신 펌웨어 (HPE Gen12 / Lenovo XCC2-3 / Dell iDRAC9 5+ / Supermicro X14+) 가 `/Power` 미응답 → 본 수집 fail.

### 3. sections enum 'not_supported' 미활용

[common/tasks/normalize/build_sections.yml](../../../common/tasks/normalize/build_sections.yml) 가 `success`/`failed`/`not_supported` enum 정의했지만 `_all_sec_unsupported` 입력 채널 없음 → 실제 'not_supported' 분류 안 됨.

### 핵심 코드 변경

**1. `_is_404_only_error` 헬퍼 + `_make_section_runner` 분리** (redfish_gather.py):
```python
def _is_404_only_error(errs):
    """모든 errors가 'HTTP 404' 시그널이면 True."""
    if not errs: return False
    for e in errs:
        detail = (e.get('detail') or '')
        msg = (e.get('message') or '')
        if 'HTTP 404' in detail or 'HTTP 404' in msg: continue
        if msg.endswith(': 404') or msg.endswith(' 404'): continue
        return False
    return True

def _make_section_runner(all_errors, collected, failed, unsupported=None):
    def _run(section, fn, *args):
        val, errs = fn(*args)
        if unsupported is not None and _is_404_only_error(errs):
            unsupported.append(section)   # noise 차단
            return val
        all_errors.extend(errs)
        ...
```
