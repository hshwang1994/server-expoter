# F03 — IPv6 수집 (Additive)

**[STATUS]**: ⚠️ **호환성 영역 외 — 새 데이터 수집** (사용자 의도 2026-05-01 명시: "호환성 fallback only"). 별도 cycle.

**우선**: P3 / **영역**: network / **출처**: R1 DMTF / **원칙**: Additive (단 새 데이터)

## 컨텍스트
EthernetInterface schema에 IPv6Addresses[] 표준. 우리 코드 IPv4만.

## 현재 동작
`redfish_gather.py:1463-1468`:
```python
ipv4_addrs = [
    {'address': a.get('Address'), 'subnet_mask': ..., 'gateway': ...}
    for a in (_safe(ndata, 'IPv4Addresses') or [])
    ...
]
```

## 변경 (Additive)
IPv6Addresses[] 추가 read. envelope `addresses[]` 에 family='ipv6' entry 추가:
```python
ipv6_addrs = [
    {'family': 'ipv6', 'address': a.get('Address'),
     'prefix_length': a.get('PrefixLength'),
     'origin': a.get('AddressOrigin')}
    for a in (_safe(ndata, 'IPv6Addresses') or [])
    if a.get('Address') not in (None, '::', '')
]
nic['addresses'] = ipv4_addrs + ipv6_addrs
```

## 회귀
- 기존 IPv4-only 환경: ipv6_addrs 빈 list → 영향 없음
- IPv6-only / dual-stack 환경: 신규 entry 추가
- schema field_dictionary: `addresses[].family` 이미 ipv4/ipv6 enum 정의됨
