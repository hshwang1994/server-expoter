**[STATUS]**: ⚠️ **호환성 영역 외 — 새 데이터 (신규 필드 추가)** (사용자 의도와 별개). 별도 cycle.

**우선**: P2 / **영역**: OS Linux storage / **출처**: R4 / **원칙**: Additive (단 새 데이터)

## 컨텍스트
현재 우리 `os-gather/tasks/linux/gather_storage.yml` 가 lsblk / df 위주. NVMe wear level / LVM (lvs/pvs/vgs) / Software RAID (mdadm/mdstat) / multipath 미수집.

## 변경 (Additive — graceful)
```yaml
# 추가 task — 도구 부재 시 skip (failed_when:false)

- name: "linux | storage | nvme list (graceful)"
  ansible.builtin.shell: nvme list -o json 2>/dev/null
  register: _l_nvme
  failed_when: false
  changed_when: false

- name: "linux | storage | LVM info (graceful)"
  ansible.builtin.shell: |
    pvs --reportformat json 2>/dev/null
    vgs --reportformat json 2>/dev/null
    lvs --reportformat json 2>/dev/null
  register: _l_lvm
  failed_when: false
  changed_when: false

- name: "linux | storage | Software RAID (graceful)"
  ansible.builtin.shell: cat /proc/mdstat 2>/dev/null
  register: _l_mdstat
  failed_when: false
  changed_when: false

- name: "linux | storage | multipath (graceful)"
  ansible.builtin.shell: multipath -ll 2>/dev/null
  register: _l_mpath
  failed_when: false
  changed_when: false
```
