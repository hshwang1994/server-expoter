# F22 — WinRM TLS 1.3 호환성

**우선**: P3 / **영역**: OS Windows / **출처**: R3 / **원칙**: 추적만

## 컨텍스트
NTLM은 신 encryption 미지원. Windows Server 2019+ / Windows 11 TLS 1.3-only 환경 시 fail 가능.

## 현재 동작
`ansible.cfg [winrm] transport = ntlm` — Windows Server 2012/2016 호환 OK.

## 향후 (사고 재현 시)
- Kerberos auth 도입 (도메인 환경)
- HTTPS 5986 + TLS 1.2 호환 펌웨어
- Python 3.8+ + urllib3 2.0.7+ (TLS 1.3 cert)

## 변경
현재 없음. 사고 재현 시 ticket 갱신.

## 회귀 risk
없음.
