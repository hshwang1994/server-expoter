# F07 — lspci 부재 환경 'not_supported' 분류

**우선**: P3 (F23 묶음) / **영역**: hba_ib / **출처**: R1+R3 / **원칙**: Additive

## 컨텍스트
minimal Alpine container / busybox / 일부 임베디드 Linux 에서 `lspci` 명령 부재. 현재 graceful (failed_when:false) 빈 list — sections.storage.hbas=[] success로 분류 (호출자 모호).

## 변경 (Additive)
`os-gather/tasks/linux/gather_hba_ib.yml`:
```yaml
- name: "linux | hba_ib | check lspci"
  ansible.builtin.shell: command -v lspci
  register: _l_lspci_check
  changed_when: false
  failed_when: false

- name: "linux | hba_ib | unsupported (lspci 부재)"
  ansible.builtin.set_fact:
    _sections_unsupported_fragment: ['hba_ib']
    _data_fragment: {}
    _errors_fragment: []
  when: _l_lspci_check.rc != 0
  no_log: true

- block:
    # 기존 lspci 수집 task
    ...
  when: _l_lspci_check.rc == 0
```

## 회귀
- 일반 RHEL/Ubuntu/Rocky: lspci 정상 → 기존 동작 유지
- minimal Alpine: lspci rc!=0 → 'hba_ib' unsupported 분류 → envelope `sections.storage.hbas` 비고 명확
