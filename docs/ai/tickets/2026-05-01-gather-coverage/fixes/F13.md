# F13 — Cisco CIMC AccountService 'not_supported' 분류

**우선**: P1 / **영역**: users / **출처**: R2 / **원칙**: Additive

## 컨텍스트
Cisco CIMC 일부 펌웨어가 AccountService 응답 미지원 또는 read-only. 현재는 `data.users=[]` + `sections.users='success'` 잘못 분류 가능 (B15 ticket).

## 현재 동작
- `/redfish/v1/AccountService` GET 시도 → Cisco 일부 펌웨어 404 / read-only / empty
- envelope `data.users=[]` + `sections.users='success'` (호출자 입장에서 "수집 OK이지만 사용자 0명"으로 오해)

## 변경 (Additive)
`redfish_gather.py` `account_service_get` 또는 wrapping task:
```python
def account_service_get(bmc_ip, username, password, timeout, verify_ssl):
    st, root, err = _get(bmc_ip, '/redfish/v1/AccountService', ...)
    if st == 404:
        # 미지원 — cycle 2026-05-01 인프라 활용
        return None, [], [{'section': 'users', 'message': 'AccountService 미지원',
                           'detail': 'HTTP 404: Not Found'}]
    # 기존 흐름 ...
```

`_make_section_runner` 가 detail에 'HTTP 404' 검출 시 unsupported 분류 (cycle 2026-05-01 자동).

또는 Cisco vendor 시그니처 매핑에 명시:
```python
# vendor='cisco' AccountService 일관 미지원이면 즉시 unsupported
if vendor == 'cisco':  # nosec rule12-r1
    # AccountService는 대부분 Cisco CIMC에서 제한
    ...
```

## 회귀
- Dell / HPE / Lenovo / Supermicro: AccountService 정상 → success 유지
- Cisco 일부 펌웨어: 404 → 'not_supported' 분류 + envelope에 section.users='not_supported'
