# F17 — Schema 버전 errata 사용

**우선**: P2 / **영역**: 횡단 / **출처**: R3 / **원칙**: Additive (검증 강화)

## 컨텍스트
DMTF Redfish-Service-Validator known issue: Service가 v1.0.0 schema 응답하는데 client가 신 schema 가정 → false validation failure. errata schema (latest patch of minor version) 사용 권장.

## 현재 동작
우리 코드 `_safe()` 가 모든 필드 read 안전 (None 반환). schema 버전 명시 없음. 신규 enum 값 (예: ProcessorType=Accelerator)은 우리 가정 위반 가능 → F02 수정 필요한 사례.

## 변경 (Additive — 검증 강화)
1. 우리 코드는 schema 버전 명시 안 함 — 변경 없음
2. 신규 enum / 필드 발견 시 ticket으로 추적 (F02 / F10 / F23 등)
3. 회귀 fixture에 다양한 schema 버전 응답 포함
