# F38 — Windows IB NIC 분류 fallback (호환성)

**우선**: P3 / **영역**: OS Windows / **출처**: R6 / **원칙**: Additive

## 컨텍스트
Windows에서 IB 카드는 IPoIB로 노출되어 normal NIC 처럼 보임. Mellanox WinOF 설치 시 IB 카드 / Ethernet 카드 / IPoIB 인터페이스 함께 존재. 우리 Windows gather 는 IB 분리 안 함.

→ 같은 IB 카드가 Linux 채널에선 `data.storage.infiniband[]` 에 분리, Windows 채널에선 `data.network.interfaces[]` 에 통합 — **채널별 일관성 부족** (호환성 영역).

## 현재 동작
`os-gather/tasks/windows/gather_network.yml`:
- `Win32_NetworkAdapter` 사용
- IPoIB / IB 카드를 Ethernet NIC 와 함께 emit (분리 없음)

## 변경 (Additive)
PowerShell `Get-PnpDevice` 로 Mellanox vendor ID 검출:
```powershell
$ib_devices = Get-PnpDevice -Class Net | Where-Object {
    $_.HardwareID -match 'VEN_15B3' -or $_.Service -match '^(mlx|ibal)'
} | Select-Object FriendlyName, HardwareID, InstanceId, Service
```

emit 시:
- Mellanox VID 매칭 NIC → `data.storage.infiniband[]` 추가 emit
- 동시에 `data.network.interfaces[]` 에도 (이미 통합 emit) — Linux 채널과 동일 패턴

## 회귀
- IB 없는 환경 (대부분 Windows host): 빈 list — 영향 없음
- IB 있는 환경 (HPC Windows): `data.storage.infiniband[]` 채워짐 — 채널 일관성
