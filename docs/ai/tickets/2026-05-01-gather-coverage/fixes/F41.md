# F41 — community.vmware 6.2.0 → 7.0.0 호환성 추적

**우선**: P2 / **영역**: ESXi / **출처**: R7 / **원칙**: Additive (호환성)

## 컨텍스트
- 우리: community.vmware 6.2.0 사용 (REQUIREMENTS.md 명시)
- community.vmware 7.0.0 에서 다수 모듈 deprecated:
  - `vmware_maintenancemode` (deprecated)
  - `vmware_vm_inventory` (inventory plugin deprecated)
  - 기타 일부
- modern alternative: `vmware.vmware_rest` (REST 기반)

## 우리 영향
`esxi-gather/` 에서 community.vmware 의 어떤 모듈 사용하는지 검토 필요:
- `vmware_host_config_info` (사용 중 — 가용 여부 확인)
- `community.vmware.vmware_*` 다수

## 변경 (Additive — 추적/계획만)
1. ansible-galaxy collection list 검증 — 우리 호스트의 모듈 목록
2. esxi-gather 모듈 사용 매트릭스 작성
3. deprecated 모듈 사용 시 alternative path 식별
4. ansible upgrade cycle 시 vmware.vmware_rest 도입 검토

## 회귀
- 현재 community.vmware 6.2.0 호환 — 즉시 영향 없음
- ansible-core 2.20+ 도입 시 deprecation warning 가능

## Cold-start
1. `ansible-galaxy collection list community.vmware` — 우리 environment
2. `grep -r "community.vmware" esxi-gather/` — 사용 매트릭스
3. ansible 11+ porting guide 추적
4. 호환성 영향 분석
