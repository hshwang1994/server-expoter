# F40 — Redfish ConnectX VPI mode 호환 (검증만)

**우선**: P3 / **영역**: network_adapters / **출처**: R6 / **원칙**: 이미 호환

## 컨텍스트
NVIDIA/Mellanox ConnectX-4/5/6/7 VPI 카드 — 같은 카드가 InfiniBand / Ethernet 양쪽 모드. mlxconfig로 LINK_TYPE 설정 (1=IB / 2=Eth/VPI).

PortType 응답 변동:
- IB 모드 → `PortType: 'InfiniBand'`
- Ethernet 모드 → `PortType: 'Ethernet'`

## 현재 동작
`redfish-gather/tasks/normalize_standard.yml:305-309`:
```jinja2
{%- set lt = 'ethernet' -%}
{%- if 'fibre' in pt or pt == 'fc' -%}{%- set lt = 'fibrechannel' -%}
{%- elif 'infin' in pt or pt == 'ib' -%}{%- set lt = 'infiniband' -%}
{%- elif 'ether' in pt -%}{%- set lt = 'ethernet' -%}{%- endif -%}
```

→ PortType raw 통과 + 정규화. **모든 PortType 변종 자동 호환**.

없음. 이미 호환.

## 검증
- ConnectX VPI 카드 IB 모드 host: `data.storage.infiniband[]` emit
- 같은 카드 Ethernet 모드 전환 후: `data.storage.infiniband[]` 빈 list, `data.network.interfaces[]` 에만 emit
- mlxconfig 변경 없이 운영 host: 응답 그대로 통과
