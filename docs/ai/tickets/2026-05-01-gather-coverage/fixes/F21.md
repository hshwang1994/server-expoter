# F21 — paramiko 2.9.0+ + RHEL 9 ssh-rsa 호환

**우선**: P2 / **영역**: OS Linux / **출처**: R3 / **원칙**: Additive

## 컨텍스트
paramiko 2.9.0+ default = rsa-sha2-512/256 우선. RHEL 9 (DEFAULT crypto policy) + 구 Linux SSH server (ssh-rsa only) 조합 fail.

증상: "no matching host key type found. Their offer: ssh-rsa"

## 현재 동작
`ansible.cfg`:
```ini
[ssh_connection]
ssh_args = -o ControlMaster=auto -o ControlPersist=60s -o StrictHostKeyChecking=no
```

→ legacy host key 옵션 없음.

## 변경 옵션 (Additive)

### 옵션 A — ansible.cfg ssh_args 확장
```ini
[ssh_connection]
ssh_args = -o ControlMaster=auto -o ControlPersist=60s -o StrictHostKeyChecking=no -o HostKeyAlgorithms=+ssh-rsa -o KexAlgorithms=+diffie-hellman-group14-sha1 -o PubkeyAcceptedAlgorithms=+ssh-rsa
```

### 옵션 B — paramiko 옵션 (Ansible inventory 변수)
```yaml
ansible_connection: paramiko
ansible_paramiko_use_rsa_sha2_algorithms: false
```

### 옵션 C — server side 업그레이드 (운영 측)
구 Linux SSH server를 RHEL 8/9 / Ubuntu 22.04+ 로 업그레이드.

## 회귀
- 옵션 A: 신 Linux server 영향 없음 (legacy 옵션 추가만 — 자동 negotiation)
- 옵션 B: paramiko 사용 시만 적용 — Ansible default는 OpenSSH 사용
- 옵션 C: 운영 영역

## 권장
사고 재현 후 옵션 A (ansible.cfg) — 단순, 모든 호스트 일관 적용

## Cold-start
1. RHEL 9 agent + 구 Linux server 사고 재현
2. 옵션 A 적용 (사용자 명시 승인)
3. 회귀 — lab 신 Linux 정상

## 회귀 risk
SSH legacy 옵션 추가 — 보안 약화 (TOFU 수준 호환). 사이트 보안 정책 확인 필수
