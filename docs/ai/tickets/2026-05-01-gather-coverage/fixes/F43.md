# F43 — RHEL 10 crypto policy SHA-1 호환성 검증

**우선**: P3 (RHEL 10 도입 시) / **영역**: TLS / **출처**: R7 / **원칙**: 검증

## 컨텍스트
- RHEL 10 DEFAULT crypto policy: SHA-1 TLS 외 허용 (RHEL 9 보다 완화)
- RHEL 9 DEFAULT: SHA-1 거의 차단 → 우리 SECLEVEL=0 + LEGACY 옵션 fallback
- RHEL 10 도입 시 우리 fallback 옵션이 RHEL 10 환경에서도 작동하는지 검증

## 우리 영향
`common/library/precheck_bundle.py:139-150` `_build_ssl_context`:
```python
if not verify:
    ctx.check_hostname = False
    ctx.verify_mode = ssl.CERT_NONE
    if hasattr(ssl, 'OP_LEGACY_SERVER_CONNECT'):
        ctx.options |= ssl.OP_LEGACY_SERVER_CONNECT
    try:
        ctx.set_ciphers('DEFAULT@SECLEVEL=0')
    except ssl.SSLError:
        pass
```

## 변경 (검증만)
RHEL 10 agent 도입 시:
1. 위 SSL context 코드가 RHEL 10 OpenSSL 3.4+ 와 호환되는지 검증
2. 구 BMC (TLS 1.0 / weak cipher) 호환성 확인
3. 새 OpenSSL legacy renegotiation 옵션 변경 추적
