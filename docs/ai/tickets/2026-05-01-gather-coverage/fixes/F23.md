# F23 — OS gather 'not_supported' 점진 전환

**우선**: P1 / **영역**: OS (Linux + Windows) / **출처**: R3 / **원칙**: Additive (인프라 활용)

## 컨텍스트
cycle 2026-05-01 fragment 인프라 (`_sections_unsupported_fragment`) 마련됨. OS gather 들도 미지원 케이스 명시 분류 가능.

미지원 케이스:
- Linux: lspci 부재 (minimal Alpine), dmidecode 부재 (일부 VM/container), getent 부재 (busybox)
- Windows: 특정 WMI Class 부재 (구 Windows Server 2008 R2)

## 현재 동작
graceful fail (failed_when:false) → 빈 list → sections.{name}='success' 잘못 분류 (호출자 모호).

## 변경 (Additive — gather 별 적용)

### Linux gather_hba_ib.yml (F07 묶음)
```yaml
- name: "linux | hba_ib | check lspci"
  ansible.builtin.shell: command -v lspci
  register: _l_lspci
  failed_when: false

- ansible.builtin.set_fact:
    _sections_unsupported_fragment: ['hba_ib']
  when: _l_lspci.rc != 0
```

### Linux gather_users.yml
이미 `getent passwd` 부재 시 'failed' 분류. 'not_supported'로 정정 검토 (P3).

### Windows gather_*.yml
WMI namespace 미지원 시 unsupported 분류. 사고 재현 후.

## 회귀
- 일반 RHEL/Ubuntu/Windows: 기존 동작 유지
- minimal container / 구 Windows: 명시 'not_supported' → envelope 호출자 친화

## Cold-start
1. gather 별로 명령 가용성 검사 추가
