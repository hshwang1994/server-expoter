# F33 — Session 인증 (X-Auth-Token) 호환

**우선**: P3 / **영역**: 인증 / **출처**: R5 / **원칙**: Additive (Basic Auth fallback 유지)

## 컨텍스트
Redfish 표준 인증 방식 2가지:
- **Basic Auth** — 매 요청 username/password (현재 우리 사용)
- **X-Auth-Token (session)** — login 1회 → token 반복 사용

5 vendor 모두 둘 다 지원 (HPE iLO 5/6 / Lenovo XCC1-3 / Supermicro X11+ / Dell iDRAC 9+ / Cisco CIMC).

성능: Basic 은 매 요청마다 verify → BMC 부하. session 은 verify 1회.

## 현재 동작
`redfish_gather.py:91-95` `_get`:
```python
req = urlreq.Request(url, headers={
    'Authorization': _auth(username, password),
    ...
})
```

매 _get / _post / _patch 호출마다 Authorization Basic.

## 변경 (Additive)
세션 생성 후 X-Auth-Token 사용. fail 시 Basic Auth fallback:

```python
def _login_session(bmc_ip, username, password, timeout, verify_ssl):
    """Redfish session login → token 반환. 실패 시 None (Basic Auth fallback)."""
    url = f'https://{bmc_ip}/redfish/v1/SessionService/Sessions'
    body = json.dumps({'UserName': username, 'Password': password}).encode('utf-8')
    req = urlreq.Request(url, data=body, method='POST', headers={
        'Content-Type': 'application/json',
        'Accept': 'application/json',
    })
    try:
        with urlreq.urlopen(req, context=_ctx(verify_ssl), timeout=timeout) as resp:
            return resp.headers.get('X-Auth-Token'), resp.headers.get('Location')
    except Exception:
        return None, None  # Basic Auth fallback

# _get / _post / _patch 가 token 있으면 X-Auth-Token 헤더, 없으면 기존 Basic Auth
```

main() 흐름:
```python
token, session_uri = _login_session(...)
if token:
    # token 기반 수집
    creds = ('token', token, ...)
else:
    # 기존 Basic Auth
    creds = (username, password, ...)

# 수집 끝나면 session DELETE (token 회수)
if token and session_uri:
    _delete_session(...)
```

## 회귀
- session 지원 BMC: token 사용 → 매 요청 latency 감소
- session 미지원 BMC (드물지만 가능): login 404 → Basic Auth fallback (기존 동작 유지)
- 동시 multi-host: 각 host별 독립 session

## 우선 P3 사유
- 우리는 단발성 gather (host당 9 endpoint × ~1초)
- Basic Auth 부하 미미 (BMC가 실제 lockout 안 거는 한)
- 현재 timeout 30초 충분
